# 项目规则 / Project Rules

## 认证系统（纯前端模拟，暂无后端）

- 登录入口为导航栏「月」右侧的「钤印」按钮；已登录后变为印章 + 笔名。
- 两套身份：骚人（访客）/ 墨客（作者）。
- 骚人：邮箱 + 前端模拟验证码（寄送朱签）。输入邮箱→点寄送→界面直接显示验证码→填入→钤印入册。
- 墨客：预发放邀请码（引帖）注册，一码一用，注册即作废；大小写不敏感。
- 状态持久化：localStorage（moke_users 用户列表、moke_session 当前登录态）。
- 核心文件 assets/js/auth.js；表单/弹窗样式在 style.css 的「钤印认证」区块。
- 后续接后端时：验证码改走邮件发送、邀请码改走服务端校验与发放、用户与 session 改走接口。
